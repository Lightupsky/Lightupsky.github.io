我从什么时候起涉猎个人博客，以及未来。

· 2022-08-13 · # 闲谈

四处为客，博览众生。

2019年末，大约是10月份，我在酷安刷到了一款初创产品【Baklib】注册送一年高级版的活动。
我想：还有这种好事？
更早的几年，追溯到小学，我就喜欢下课拿本本子乱涂乱画，写点有的没的。在小学五年级的时候，我在一节漫长而无事的自修课上写了一篇关于怎么给Android Root的文章，我接触搞机比这更早一些，大概在2016年，那时候酷安V7还未正式发布——现在已经是V12了。
因此，我便早有写点文章的习惯，也有分享、传播的意愿，借着这个契机，在Baklib创建了第一个线上博客。在那一年时间里，我写了多少呢？
四篇😀
你不能对一个拖延习惯的人有着过高的要求，更何况并没有什么人看（其实也找不到）我写的内容。年纪小啥都喜欢锐评一嘴，我记得有写关于996的，关于227的，关于丁真的...阅历浅薄的我自然给不出多少真知灼见，只是一些无谓的“冲塔”泄愤，当热情过去，大海退潮，只剩一地鸡毛。
今日我在必应搜索，还能窥见那时文章的标题：

初创企业的承诺，就像林力健大师的音道一样善变。在这篇发布于2019年11月20号的知乎文章中，Baklib承诺在2020年前的注册用户终身享有企业版免费，然而现在，企业版仅限单域名，更高级的付费套餐，需要绑定自己的域名才保证稳定性等，我理解这是企业的权宜之计，但这也让我决定不再使用这些博客托管平台。

我要自己搭个博客网站！
关于个人博客，一直搁置到2022年1月23日，才有了动静。通过油管的一个教学视频（电脑更换，收藏链接迷失在了硬盘里），我用一晚上时间借助开源博客主题Contrast和GithubPages上线了新的博客。在我Github仓库的master分支就可以看到当时博客的内容。
生命贵在折腾，几个痛点让我最终没有坚持这么运作：

1.Markdown语法“太规范了”

这的确是我的问题，我的所有文章都保存在【纯纯写作】APP上，因为当时我在手机端的写作效率更高，并且接触手机要方便得多，在Markdown模式下我依然是回车换行。但我没有注意到设置里的可选项，也没有特意查看Markdown的文档。这个疏忽为我发布本地的文章增加了很多工作量，当我把文章复制到Github中，我惊讶地发现，预览时的文章一个换行都没有。
引用Markdown官方文档中的一段话：

几乎每个 Markdown 应用程序都支持两个或多个空格进行换行，称为 结尾空格（trailing whitespace) 的方式，但这是有争议的，因为很难在编辑器中直接看到空格，并且很多人在每个句子后面都会有意或无意地添加两个空格。由于这个原因，你可能要使用除结尾空格以外的其它方式来换行。幸运的是，几乎每个 Markdown 应用程序都支持另一种换行方式：HTML 的 <br> 标签。

每篇文章都要重新排版，懒癌犯了，并且的确没有产出什么新内容，这是其一。

2.繁琐的内容更新流程

现在选择Gridea编辑博客的最大原因就是，当我写完一篇，只需轻点一下左下角的同步按钮，我的文章就会在1分钟内更新到我的网页。我只管写就是了。
但在之前，如果我写完了一篇文章，那我会手动上传.md文件，编辑文件名为主题可读取的格式，编辑内容增加标签和内容标识（post/layout...），检查无误后push并等待Action构建完毕。

3.所见非所得的开盲盒式编辑

看完在本地搭建博客环境，通过localhost预览博客页面的教程后，我选择了放弃...
当然的确可以通过使用git push，对先前的文章查找替换规范格式，如上述所说搭建环境，但我只是写个文章，也许有点本末倒置。

这次的改造大概会稳定一段时间了，我也会开始产出一些新的内容，把之前相对满意的内容发上来。
关于这个博客的未来，就像我的人生一样，充满了不确定性，随遇而安。
也许哪天就关站跑路（
